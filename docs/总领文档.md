# AIJu 总领文档

## 1. 项目目标

AIJu 面向“学科知识整合智能体开发”赛题，目标是在 5 小时黑客松节奏内交付一个可演示、可复现、可扩展的 Web 系统：

- 多格式教材上传与解析，首要支持 PDF / Markdown / TXT。
- 单本教材知识点抽取、关系识别与可视化知识图谱。
- 跨教材语义对齐、合并去重、压缩到原始体量 30% 以内。
- 基于教材 chunk 的 RAG 问答，每个回答必须带教材、章节、页码引用。
- 教师通过多轮对话调整整合决策，并在图谱与报告中反映。
- 输出完整开发文档、Agent 架构说明和整合报告。

当前仓库只放工程与文档。赛题 PDF 和 7 本教材保留在仓库外层，默认通过 `TEXTBOOK_DATA_DIR=../textbooks` 读取。

## 2. 本地事实

本地测试教材共 7 本，约 2567 页 / 822.8MB：

| 文件 | 页数 | 大小 |
|---|---:|---:|
| 01_局部解剖学.pdf | 305 | 111.5MB |
| 02_组织学与胚胎学.pdf | 319 | 417.6MB |
| 03_生理学.pdf | 450 | 31.4MB |
| 04_医学微生物学.pdf | 386 | 32.6MB |
| 05_病理学.pdf | 418 | 38.5MB |
| 06_传染病学.pdf | 398 | 127.8MB |
| 07_病理生理学.pdf | 291 | 63.4MB |

结论：解析必须逐页进行，索引和知识抽取必须分批缓存；全量 LLM 抽取需要限速、断点续跑和章节级结果落盘。

## 4. 模块实施路线

1. 教材解析
   - 上传文件写入 `data/uploads/`，记录 `TextbookSummary`。
   - PDF 使用 PyMuPDF 逐页解析；MD/TXT 直接读取；DOCX/Excel 后续由 MarkItDown 转 Markdown。
   - 通过正则 `第X章`、字号变化和页码连续性识别章节。
   - 输出统一结构：教材 ID、文件名、总页数、总字数、章节列表。

2. 知识图谱构建
   - 以章节为最小 LLM 调用单元，Prompt 强制 JSON 输出。
   - 节点字段：名称、定义、类别、章节、页码、原文片段。
   - 边类型至少支持 `prerequisite`、`parallel`、`contains`、`applies_to` 中的三种。
   - JSON 经 Pydantic 校验，不合格则重试一次，仍失败则标记章节失败。

3. 跨教材整合
   - 对节点名称、定义、同义词进行规范化。
   - 用 BGE embedding 召回相似节点候选，阈值初始设为 0.82。
   - 对 0.78-0.90 的临界候选调用 LLM 判断是否等价。
   - 输出 `merge / keep / remove` 决策，保留理由和置信度。
   - 按出现频次、关系中心性、定义完整度和章节覆盖度选择合并后的最佳表述。

4. RAG 问答
   - chunk 大小 500-800 字，默认 650 字，overlap 默认 80 字。
   - 每个 chunk 保留教材名、章节、页码、字符范围。
   - Chroma 持久化向量与元数据。
   - 查询 top-5 chunk，Prompt 限定只基于上下文回答。
   - 无答案时返回“当前知识库中未找到相关信息”。

5. 教师反馈闭环
   - 右侧对话面板记录同一会话历史。
   - 支持自然语言修改：保留、拆分、合并、恢复删除。
   - 修改后重新计算整合图谱和报告统计。

## 5. API 总览

| Endpoint | 目的 |
|---|---|
| `POST /api/textbooks/upload` | 上传教材 |
| `GET /api/textbooks` | 获取教材列表和解析状态 |
| `POST /api/textbooks/{id}/parse` | 解析单本教材 |
| `POST /api/graph/build` | 构建知识图谱 |
| `GET /api/graph` | 获取当前图谱 |
| `POST /api/integration/run` | 运行跨教材整合 |
| `POST /api/integration/feedback` | 应用教师反馈 |
| `POST /api/rag/index` | 构建向量索引 |
| `POST /api/rag/query` | 带引用问答 |
| `GET /api/rag/status` | 索引状态 |
| `GET /api/report/summary` | 报告统计摘要 |

## 8. 风险与降级

- LLM 调用慢：只抽取每章前 N 个 chunk，缓存结果，演示时优先 1-2 本教材。
- PDF 质量差：先使用文本层；无法抽取时标记失败并提示后续 OCR。
- 全量教材过大：首版使用异步任务和进度状态，报告可先基于外部教材抽样生成。
- embedding 模型下载失败：切换到 sentence-transformers multilingual MiniLM 或云端 embedding。
- Cytoscape 节点过多：默认按教材/章节/频次过滤，只渲染 top 节点。
