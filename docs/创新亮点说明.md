## 1. 多 LLM 提供商容错链 + 离线 Embedding 降级

**问题**：黑客松环境网络不稳定、GPU 不可用、API Key 可能失效。传统方案依赖单一路径，任意环节故障即导致全链路不可用。

**方案**：AIJu 实现了三级自动降级链。

| 层级 | 路径 | 触发条件 | 效果 |
|------|------|----------|------|
| 优先 | Anthropic API（mimo-v2.5-pro） | `ANTHROPIC_AUTH_TOKEN` 已配置 | 全功能 RAG 对话 |
| 备用 | OpenAI-compatible API（DeepSeek） | `OPENAI_API_KEY` 已配置 | RAG 回答 + 整合 LLM 判定 |
| 降级 | Hash 伪向量（blake2b） | `sentence-transformers` 未安装 | 无 GPU 仍可检索和整合 |

实际效果：

- **断网场景**：hash 向量 + BM25 混合检索 + 规则答案组合，系统可完成基本问答和整合决策展示。
- **无 GPU 场景**：`BAAI/bge-small-zh-v1.5` 安装失败时自动降级到 blake2b，不阻塞索引构建。
- **API Key 过期**：Anthropic 不可用时自动切换到 OpenAI-compatible 路径。

**技术细节**：

```python
# rag_service.py: _get_bge_model()
def _get_bge_model():
    try:
        from sentence_transformers import SentenceTransformer
        return SentenceTransformer(settings.embedding_model)
    except Exception:
        return False  # → 自动切换到 hash 伪向量
```

**文档位置**：`docs/Agent 架构说明.md` §8、`docs/系统设计.md` §2。

## 2. 教师反馈闭环 —— AI 初稿 + 教师定稿

**问题**：传统知识整合系统是黑箱输出，教师无法参与决策。

**方案**：AIJu 实现了「AI 初稿 → 教师审核 → 反馈修改 → 重新整合」的闭环工作流。

**交互方式**：

- 教师通过右侧「对话」面板输入自然语言指令（如"把抗原和免疫原分开"）。
- Feedback Tool 通过关键词解析（拆分/保留/删除/合并）转换为具体决策指令。
- 系统重新计算整合图谱和报告统计。

**API**：`POST /api/integration/feedback`，请求体：

```json
{
  "session_id": "default",
  "decision_id": "merge_001",
  "message": "把抗原和免疫原分开，它们不是同一个概念"
}
```

**效果**：教师可以在对话历史中追溯所有修改，每条反馈都有对应决策 ID 和效果。

## 3. 知识图谱形状-颜色-大小三维视觉映射

**问题**：赛题要求「多维度叠加、层次丰富」的图谱可视化。

**方案**：AIJu 的 Cytoscape 图谱同时使用三个视觉维度：

| 维度 | 映射 | 示例 |
|------|------|------|
| 颜色 | 教材来源（6 色调色板） | 局部解剖学=绿色，组织学=紫色 |
| 形状 | 知识点类别 | 机制=圆角矩形，方法=菱形，分类=六边形 |
| 大小 | 跨教材出现频次 | 高频概念节点更大 |

**交互增强**：

- **悬停 tooltip**：鼠标悬停节点显示名称 + 定义前 60 字。
- **双击展开邻居**：双击某节点自动高亮 1-hop 邻域，其余节点淡化。
- **搜索筛选**：输入关键词实时高亮匹配节点。

## 5. 双通道混合检索（向量 + BM25）

**问题**：单独向量检索对医学术语的精确匹配能力不足，单独 BM25 无法捕捉语义相似概念。

**方案**：AIJu 使用加权混合检索：

```text
final_score = 0.68 × cosine_similarity + 0.32 × normalized_bm25_score
```

**效果**：

- 语义检索能召回"炎症"相关但用词不同的概念（如"炎性反应"）。
- BM25 确保精确术语匹配不丢失（如"肽聚糖"只出现在特定章节）。

## 6. 配置的 DeepSeek V4 Flash 后端

**当前配置**：

- 主 LLM：Anthropic 兼容代理 `mimo-v2.5-pro`（RAG 对话）
- 备用 LLM：DeepSeek V4 Flash `deepseek-chat`（整合 LLM 判定 + RAG 回答备用）
- Embedding：`BAAI/bge-small-zh-v1.5` + blake2b hash fallback

**DeepSeek 接入验证**：

```bash
curl https://api.deepseek.com/v1/chat/completions → 200 OK, 0.78s
```

模型实际路由为 `deepseek-v4-flash`（最新 V4 架构），兼容 OpenAI Chat Completions API。
