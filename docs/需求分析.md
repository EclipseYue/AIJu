# 需求分析

## 1. 用户与场景

主要用户是学科教师和教学管理者。他们希望快速理解多本教材之间的重复、互补和缺失内容，并得到一份压缩后的教学知识版本，同时仍能追溯到教材原文。

核心场景：

- 教师上传多本教材，查看每本教材的章节和知识结构。
- 教师查看跨教材重复知识点，确认系统为什么合并或删除。
- 教师提出问题，系统基于教材原文回答并引用来源。
- 教师通过对话修改整合方案，系统重新更新图谱和报告。

## 2. 知识点粒度

首版知识点定义为“可独立教学、可被问答引用、可与其他知识点形成依赖关系的概念单元”。医学教材中的典型粒度包括：

- 概念：炎症、动作电位、白细胞、免疫应答。
- 机制：凝血过程、病原体传播机制。
- 分类：细胞类型、疾病分类、病理变化类型。
- 方法/诊断：染色方法、实验检测方法。

不把普通段落标题、纯例题、页眉页脚、参考文献作为知识点。

## 3. 重复判定标准

重复知识点必须满足以下至少两项：

- 名称高度相似或存在明显同义词、英文/中文对应关系。
- 定义语义相似，embedding 相似度超过阈值。
- 所属上位概念一致，关系邻域相似。
- LLM 判断为同一教学概念。

默认流程：

1. 名称归一化：去空格、统一大小写、括号中英文拆分、常见医学同义词表。
2. embedding 召回：BGE 计算节点名称 + 定义向量，相似度大于 0.82 直接进入候选。
3. LLM 校验：对 0.78-0.90 的临界候选输出“等价 / 相关但不等价 / 不相关”。
4. 人类可覆盖：教师反馈优先级高于系统自动判断。

## 4. 教学连贯性保障

压缩不是简单删除低频内容，而是保持教学路径：

- 前置依赖链不能断裂，若保留高级概念，必须保留其必要前置概念。
- 包含关系中若删除下位概念，需要在上位概念摘要中保留必要说明。
- 高频知识点优先保留，但低频的关键桥接节点不能自动删除。
- 每个删除或合并决策必须保留理由和来源，便于教师复核。

## 5. 压缩比计算

压缩比使用字符数口径：

```text
compression_ratio = integrated_chars / original_chars
```

`original_chars` 为所有解析章节正文的总字符数，不包含页眉页脚、目录、参考文献等非正文内容。`integrated_chars` 为整合后知识点定义、保留章节摘要和必要教学说明的字符数。验收目标为 `compression_ratio <= 0.30`。

## 6. RAG 分块策略

默认 chunk 大小为 650 个中文字符，overlap 为 80 个字符。

理由：

- 500-800 字能容纳一个医学概念的定义、机制和例子。
- 80 字 overlap 可以降低跨页或跨段落截断风险。
- top-5 chunk 注入 LLM 时上下文长度可控。
- chunk 元数据保留教材、章节、页码，满足引用要求。

后续 Benchmark 对比：

| 策略 | 指标 |
|---|---|
| 300 字 chunk | 检索精度、上下文碎片化程度 |
| 650 字 chunk | 默认基线 |
| 900 字 chunk | 跨段完整性和 token 成本 |
| 向量检索 + BM25 | 召回率提升 |
| Rerank | 引用准确率提升 |

## 7. 优先级

P0 必做：

- PDF/MD/TXT 解析。
- 知识点抽取和图谱 JSON。
- Cytoscape.js 交互图谱。
- 跨教材整合决策和 30% 压缩统计。
- RAG 问答引用来源。
- 教师反馈修改决策。
- README、需求分析、系统设计、Agent 架构说明、整合报告。

P1 加分：

- DOCX/Excel 支持。
- 混合检索 + Rerank。
- 多视图图谱。
- Docker 一键部署。
- RAG Benchmark。
