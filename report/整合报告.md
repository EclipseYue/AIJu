## 4. 重点整合案例

### 案例 1：跨教材 CIP 信息合并

- **涉及教材**：01_局部解剖学、02_组织学与胚胎学、04_医学微生物学
- **涉及知识点**：「图书在版编目」（CIP 数据）
- **系统决策**：合并（merge）
- **决策理由**：名称完全相同（相似度 1.00），三本教材的版权页均含 CIP 编目信息，属于 PDF 元数据冗余，保留定义更完整的版本。
- **教师反馈**：无（系统自动判定，置信度 1.0）

### 案例 2：相同知识点名称精确匹配

- **涉及教材**：01_局部解剖学、02_组织学与胚胎学
- **涉及知识点**：「图书在版编目」
- **系统决策**：合并（merge）
- **决策理由**：名称/定义语义相似（相似度 1.00），保留定义更完整的版本。两个节点来自不同教材但表示完全相同的概念。
- **教师反馈**：无（系统自动判定）

### 案例 3：嵌入相似度触发合并

- **涉及教材**：跨教材
- **涉及知识点**：多个核心概念节点
- **系统决策**：合并（merge）
- **决策理由**：通过 hash 伪向量计算 cosine 相似度 ≥ 0.82 的节点对自动合并，`pick_winner` 算法基于频次 ×2 + 定义长度 ×0.01 + 关联边数 ×1.5 选择保留节点。
- **教师反馈**：无（建议在完整 API 配置下启用 LLM 临界判定，对 0.78-0.82 区间的节点对进行语义等价验证）

## 5. 教学完整性说明

- **前置依赖链保留**：如「局部解剖学的学习目的和学习方法」→「人体的分部、层次和基本结构」→「解剖器械的准备和使用」，三层递进关系完整保留。
- **冗余覆盖**：合并操作基于名称精确匹配和嵌入相似度，确保被合并的知识点被更完整版本覆盖。
- **跨教材知识缺口**：当前处理的前言/目录章节以元信息为主，尚未发现教学概念性缺口。完整文本处理预计会发现更多跨教材互补知识点。
- **教师反馈机制**：`/api/integration/feedback` 端点支持教师对任意决策输入反馈（拆分/保留/删除/合并），系统自动应用并重建整合图谱。

## 6. 生成方式

本报告数据来源于以下系统组件：

- `TextbookParserService`：PDF 解析与章节切分
- `GraphBuilderService`：LLM 知识点提取与知识图谱构建
- `IntegrationService`：三阶段跨教材整合（名称归一 + embedding 相似度 + LLM 临界判定）
- `ReportService`：整合数据聚合与报告摘要
- `RagService`：RAG 索引、混合检索与 LLM 答案生成

完整 API 调用的数据流：

```
POST /api/textbooks/{id}/parse → POST /api/graph/build → POST /api/integration/run → GET /api/report/summary
```
